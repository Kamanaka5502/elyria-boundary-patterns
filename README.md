# Elyria Systems — Boundary Patterns

Public pattern catalog for admissibility, refusal, replay, standing, authority, and consequence-boundary enforcement.

This repository is a public-safe pattern surface. It does not contain proprietary runtime logic, resolver internals, evaluator algorithms, private thresholds, production adapters, or customer-specific policy material.

## Purpose

Boundary patterns make Elyria Systems legible to buyers without exposing the engine.

Each pattern describes:

- the risk condition
- the boundary question
- the admissibility check
- the lawful outcome
- the proof artifact
- what remains private

## Pattern families

- stale evidence refusal
- privileged access boundary
- authority revocation
- receipt-bound continuation
- replay verification
- degraded standing
- no hidden continuation after breach

## Foundation lineage
